# 1000mu市场运营工作台

千目科技自媒体运营全流程工作台。1人团队专用，覆盖从选题到发布到归档的完整链路。

## 核心定位

- **目标用户**：千目科技市场运营（1人编制）
- **运营平台**：小红书、头条号、网易号、知乎、百家号、搜狐号、B站（共7个）
- **使用频率**：工作日每天启动一次
- **核心诉求**：在人力极有限的前提下实现高效内容产出与留资转化

## 产品知识来源

执行任务时，**必须先调用 `miot-product-knowledge` skill** 获取千目产品知识库信息，包括：
- 核心产品线：指路机、AI导购智能体、智能导视系统、智能标识、AR导航等
- 典型项目案例
- 目标客户画像（商业地产商、场馆运营方、园区管理方）
- 竞品差异化卖点

## 模块一：每日启动 — 选题规划

### 触发条件
工作日每天启动时自动执行。

### 选题原则

1. **产品线轮换**：每周在以下主题中轮换排期，避免连续两天推同一产品：
   - AI导购智能体 / 千目购小程序
   - 智能导视系统
   - 智能指路机
   - 智能标识
   - AR导航
   - 整合方案类（多产品组合）

2. **场景多样化**：每个产品下可从不同使用场景切入：
   - 购物中心/商场
   - 园区/写字楼
   - 展馆/博物馆
   - 医院/交通枢纽
   - 文旅景区

3. **风格多样化**：避免一直用同一种写法，主动切换风格：
   - 干货科普风（主基调）：行业知识科普 + 产品解决方案
   - 行业案例风：以具体项目案例切入讲故事
   - 数据/趋势风：引用行业数据引出需求
   - 痛点问答风：以常见客户问题为切入点
   - 对比分析风：传统方案 vs 千目方案对比

4. **查重检查**：参考近期已发布的选题记录，避免题材和角度过于接近。

### 输出
向用户提供当日选题建议（含推荐风格和切入场景），等待确认后进入内容生产环节。

## 模块二：内容生产

### 触发条件
确认选题后自动执行。

### 双版本稿件规则

| 维度 | 小红书版 | 其他6平台版 |
|------|---------|------------|
| 平台 | 小红书 | 头条、网易、知乎、百家号、搜狐、B站 |
| 风格 | 更轻松活泼、可适当用emoji、段落更短 | 偏专业干货、结构更完整 |
| 字数 | 300-600字 | 500-1000字 |
| 标题 | 吸眼球、可带emoji | 专业准确、利于SEO |

### 内容要求（所有版本通用）

1. **文字精炼不冗长**：不要凑字数，每句话都要有信息量
2. **必须包含引流引导**：每篇文章结尾或自然位置植入咨询引导，例如：
   - "想了解您的场地适合什么方案？欢迎私信交流"
   - "有导视/导航需求的商场或园区可以联系我们做免费方案评估"
   - "点击主页了解更多案例"
3. **体现专业度**：基于 miot-product-knowledge 的准确产品信息撰写，不编造功能
4. **标题吸引力**：标题要能引起目标读者（商业地产/场馆运营方）点击欲望
5. **SEO友好**（其他6平台版）：合理布局关键词，便于搜索收录

### 输出
生成两版完整稿件文本供用户审阅。

├── 智能体/
├── 指路机\

### 匹配逻辑
1. 根据当篇文章的主题和场景，从对应子文件夹中挑选最匹配的项目案例图
2. 如果素材库中该分类下没有图片，明确告知用户需要补充哪类图片
3. 提示用户图片需要带公司水印后使用

### 注意事项
- 用户部分素材存储在钉钉钉盘中，当前无法直接访问钉盘
- 如需钉盘中的素材，请提示用户下载到本地素材库对应文件夹
- 用户随时可能补充新素材到文件夹，每次操作时重新扫描目录

## 模块四：发布执行

### 触发条件
稿件确认后，用户已打开浏览器并登录各平台后台。

### 发布流程
1. **确认用户准备就绪**：确保用户已在浏览器中登录了需要发布的平台
2. **按平台逐一发布**：使用浏览器自动化能力，将对应版本稿件填写到各平台发布界面
3. **配图上传**：同步上传匹配好的配图
4. **发布确认**：每完成一个平台的发布，反馈状态

### 发布对照表
| 平台 | 使用版本 | 说明 |
|------|---------|------|
| 小红书 | 小红书版 | 单独发布 |
| B站 | 其他6平台版 | 可配视频封面 |
| 头条号 | 其他6平台版 | |
| 网易号 | 其他6平台版 | |
| 知乎 | 其他6平台版 | |
| 百家号 | 其他6平台版 | |
| 搜狐号 | 其他6平台版 | |

### 异常处理
- 如某平台发布失败，记录错误原因并跳过，最后汇总报告给用户
- 如验证码/安全验证拦截，暂停并通知用户手动处理后继续

本周完成工作：

2、[具体事项2]：[详细说明]
3、[具体事项3]：[详细说明]
4、[具体事项4]：[详细说明]

需协调工作：

[如有需要协调的事项列出，无则填"无"]

## 模块六：月度归档

### 触发条件
**每月月初**（每月1-3个工作日内），统计整理上个月的内容。或用户指定时间触发。

### 归档前准备 — 公司表格格式确认
1. **首次使用时**：请用户提供公司要求的月度归档表格模板/格式截图
2. **后续月份**：按已确认的格式模板自动套用
3. 如公司格式有变更，用户可随时提供新的模板更新

### 公司表格格式（已确认 ✅）

**文件名格式**：`XX-自媒体内容发布归档备案表（YYYY年M月）.xlsx`

**表头信息**：
- 表名：`<千目信息>-自媒体内容发布备案登记表`
- 编号：`ZMTDJ01 - 自媒体内容发布备案登记表`
- 发布周期：`2026年______月`（手动填写）
- 备案部门：`市场部`
- 说明：每月最后一个工作日下班前，将当月自媒体发布备案登记表上传至Nas系统中的指定文件夹

**数据列字段（共10列）**：

| 列号 | 字段名 | 说明 | 示例 |
|------|--------|------|------|
| 1 | 序号 | 自动编号 | 1, 2, 3... |
| 2 | 发布日期 | 实际发布日期 | 2026/3/9 |
| 3 | 内容标题 | 文章标题 | 商场停车难怎么解决？千目智慧导车方案给出完美答案 |
| 4 | 发布账号 | 发布平台账号名称 | 千目信息搜狐号、千目信息微信公众号等 |
| 5 | Url | 文章链接 | https://www.sohu.com/a/... |
| 6 | 备注 | 补充说明（如"同步千目信息其他自媒体账号"） | 可选填 |
| 7 | 审核级别 | 固定值 | 一级审核 |
| 8 | 最终审核通过日期 | 登记时的日期 | 2026/4/9 |
| 9 | 登记日期 | 填表日期 | 2026/4/9 |
| 10 | 登记人 | 固定值 | 谢媛媛 |

### 归档内容
1. **上月发布文章汇总表**：按上述公司格式生成Excel文件
   - 数据来源：上月每天发布的所有平台文章记录
   - 文件命名：`05-自媒体内容发布归档备案表（YYYY年M月）.xlsx`
   - 注意：Url列小红书和部分平台可能无直接链接，如实填写即可；B站如发的是视频则填视频链接
2. **每篇文章Word化**：将上月发布的每一篇内容转换为Word文档
   - 包含：标题、正文、配图、发布平台信息、发布日期

### 输出路径
建议输出到 `F:\1000mu-media\归档\YYYY年MM月\`

### NAS上传
- NAS上传由用户自行操作
- skill 负责整理好所有文件并告知用户需要上传的内容清单

## 注意事项

1. **用户是1人团队**：所有产出要实用高效，避免不必要的中间步骤
2. **引流优先**：每篇内容都要考虑留资转化效果
3. **素材库渐进建设**：初期素材可能不足，逐步积累，每次提醒用户补充
4. **灵活应变**：用户可能有临时插入的任务（如POC项目、展会配合），优先响应用户指令
5. **产品信息准确性**：涉及产品功能和参数时务必以 miot-product-knowledge 为准
