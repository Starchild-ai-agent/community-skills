# Crypto Content Writing Style Guide

## Voice & Tone

Write with quiet confidence — the kind that comes from knowing the product works. Technical but never inaccessible, direct but never aggressive. Sound like builders talking to builders, not marketers chasing clicks.

Adapt to the brand's personality when provided. A consumer app can be warmer. An institutional product should be more measured. A dev tool can be blunter. The rules below apply regardless.

What to avoid: desperation, hype, vagueness, anything that sounds like you're trying too hard.

## Sentence Flow

**The most important rule: write like a human, not like AI.**

Avoid short, choppy sentences that feel robotic or list-like. Instead of breaking every thought into its own sentence, combine related ideas into longer sentences that flow naturally from one point to the next.

Bad (AI-sounding):
> "The system is efficient. It saves gas. Users benefit from lower costs. This improves the experience."

Good (natural flow):
> "The system is more efficient because it batches transactions together, which means users pay less gas and get a better experience overall."

**Never use em dashes.** They're an AI tell. Use commas, parentheses, or restructure the sentence instead.

Bad:
> "The vault—which manages liquidity automatically—reduces the need for manual intervention."

Good:
> "The vault manages liquidity automatically, which reduces the need for manual intervention."

**Never use horizontal dividers (---) between sections.** They make content look AI-generated. Let headers do the work of separating sections.

## Word Choices

Swap these out:

| AI-sounding | Human-sounding |
|-------------|----------------|
| excited, thrilled | confident, ready |
| revolutionary | improved, better |
| game-changing | significant |
| utilize | use |
| leverage (verb) | use |
| in order to | to |
| it's worth noting that | (just say it) |
| importantly | (just say it) |
| seamless | (just describe what it does) |
| robust | (just describe what it does) |

Hard avoid: exciting, massive, huge, moon, WAGMI, LFG, best-in-class, world-class, synergy, ecosystem play, next-generation.

## Technical Explanations

Start with context so the reader understands why they should care, then introduce the technical concept, then explain how it works and what benefit it provides.

Bad:
> "We use concentrated liquidity for better capital efficiency."

Good:
> "Traditional AMMs spread your capital across every possible price, even prices that will never get hit. Concentrated liquidity lets you focus that capital in the ranges where trading actually happens, so the same dollar works harder."

## Numbers & Data

Use specific numbers when you have them. Round to meaningful precision ($1.2M not $1,234,567). Compare to benchmarks when it helps the reader understand scale.

## Formatting

**Articles:** Use H1 for the title only, H2 for main sections, H3 for subsections. Bold key terms on first use. Use bullet lists when you have three or more items that are genuinely parallel — not as a way to avoid writing prose.

**Tweets:** Break lines for readability. No hashtags. Use @ for partner tags. Write numbers as digits. Contractions are fine.
